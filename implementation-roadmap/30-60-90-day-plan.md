# 30/60/90-Day Modernization Plan

## Purpose

This plan outlines a phased 90-day approach for helping Contoso Operations begin its Azure modernization journey and prepare for production-ready AI workloads.

The plan is designed to create early progress while reducing risk, improving governance, and building a foundation for scalable adoption.

## First 30 Days: Assess and Prepare

### Objectives

- Confirm business goals and modernization priorities
- Assess current-state systems, data sources, workflows, and risks
- Establish cloud governance and security baseline
- Identify the first pilot workload

### Key Activities

- Conduct stakeholder discovery sessions
- Inventory legacy SQL Server, Excel, SharePoint, and shared drive assets
- Identify business-critical workflows
- Review identity and access patterns
- Define tagging, ownership, and cost management standards
- Identify monitoring and alerting requirements
- Select one modernization pilot use case

### Deliverables

- Current-state assessment
- Usage-vs-needs analysis
- Blockers and risks list
- Initial governance recommendations
- Pilot workload selection

## Days 31-60: Build Foundation and Modernize Pilot

### Objectives

- Establish the Azure foundation for the pilot
- Modernize one priority workflow or data source
- Configure monitoring, security, and access controls
- Validate early business value

### Key Activities

- Configure resource group and naming standards
- Apply RBAC and least privilege access
- Define Key Vault and managed identity pattern
- Modernize one priority data source into Azure SQL Database
- Configure Azure Monitor and Application Insights
- Build Power BI reporting view for pilot metrics
- Document support and ownership model

### Deliverables

- Pilot architecture
- Modernized data source or workflow
- Monitoring baseline
- Security and access model
- Initial reporting dashboard
- Lessons learned

## Days 61-90: AI Pilot and Scale Plan

### Objectives

- Introduce one controlled AI use case
- Validate AI readiness using approved data sources
- Establish governance and review process
- Define scale roadmap

### Key Activities

- Select approved documents for AI retrieval
- Index content using Azure AI Search
- Configure controlled AI assistant pattern
- Define prompt, response, and human review process
- Monitor usage, latency, cost, and feedback
- Review pilot outcomes with stakeholders
- Prioritize next workloads for modernization

### Deliverables

- AI pilot plan
- Approved data source inventory
- AI governance checklist
- Usage and feedback review
- Scale roadmap
- Executive summary

## Success Criteria

By the end of 90 days, Contoso Operations should have:

- A documented cloud modernization roadmap
- One pilot workload modernized or prepared for modernization
- Foundational security and monitoring controls defined
- A controlled AI pilot design
- Clear ownership and support model
- Prioritized next steps for broader adoption
